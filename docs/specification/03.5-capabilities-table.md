### 3.5 Capabilities

The table below enumerates CORTEX's 33 capabilities across five categories: Infrastructure, Measurement, Analysis, Validation, and Future/Advanced. Each capability traces to the user stories and principles it enables. The Rationale column is a one-sentence summary; full justifications with related work citations are provided in Appendix B.

**Status legend:** Yes = Implemented, Partial = Needs Enhancement, No = Not Yet Implemented.

#### Infrastructure

| Capability | Enables | Status | Rationale (see Appendix B for detail) | Strategy |
|-----------|---------|--------|---------------------------------------|----------|
| Oracle validation | P2; SE-1,2,3, AR-1, HE-1,2 | Yes | Validates C kernel output against Python/SciPy reference (`rtol=1e-5`, `atol=1e-6`) before any benchmark run. | Implemented |
| Coordinated omission resistance | P1; SE-1,6,7 | Partial | Constant-rate window replay prevents measurement backoff during stalls; HdrHistogram correction algorithm not yet integrated. | Adapt |

| Transports (TCP, Serial, Local) | SE-1 | Yes | Unified `cortex_transport_t` interface with URI-driven selection; same protocol across local, TCP, and serial transports. | Implemented |

#### Measurement

| Capability | Enables | Status | Rationale (see Appendix B for detail) | Strategy |
|-----------|---------|--------|---------------------------------------|----------|
| Sustained measurement | P1; SE-6 | Yes | Default 120s × 5 repeats = 600s total (~1,100 windows at 2/sec after warmup) for reliable percentile estimation. | Implemented |
| Warmup protocol | P1; SE-6 | Yes | Discards first N warmup windows (default 10s) to filter cold-start transients (cache, DVFS, thermal ramp). | Implemented |
| Load profiles | P3,P4; SE-4 | Yes | Three-tier declarative profiles (idle/medium/heavy) spawn stress-ng; medium load prevents DVFS downscaling on locked-down platforms. | Implemented |
| Two-phase measurement | P1; SE-6 | Yes | ABI enforces init/process/teardown separation; allocations in `process()` are contract violations, not just discouraged. | Implemented |
| Platform-state capture | P4; SE-5,7 | Partial | Thermal capture via sysfs implemented. PMU-derived per-window effective frequency and frequency tax available via `cortex decompose` (cycle count / wall time). Per-window osnoise captured on Linux via tracefs. Sysfs-based per-window CPU frequency, governor state, and compiler flags still missing from raw telemetry records. | Adapt |
| Statistical confidence | P1; SE-6,7 | Partial | 1,200 windows per run with full distribution stats; missing explicit CI reporting (mean ± CI at 95%). | Adapt |
| Multi-dtype kernels | P2; SE-3 | Yes | ABI defines FLOAT32, Q15, Q7 as first-class types. 9 kernels in f32, 8 in Q15 (welch_psd excluded from Q15—requires Q15 FFT internally, but standalone FFT kernel now available in both dtypes). Q15 oracles model exact integer arithmetic chains; all validated C-vs-oracle within quantized tolerances (rtol=1e-3, atol=1e-3; FFT Q15 uses relaxed rtol=5e-2 due to C_FIXDIV scaling approximation in oracle). Infrastructure: dtype-aware scheduler buffers, replayer float32→Q15 conversion, dtype in telemetry records. Q7 (int8_t) deferred—same infrastructure applies, but higher quantization noise requires separate validation study. | Implemented |
| Performance counters | P4,P5; SE-5,7 | Partial | PMU instruction count, cycle count, and backend stall cycles captured via kpc (macOS) and perf_event (Linux). `cortex decompose` derives IPC, effective frequency (GHz), frequency tax (%), and compute/memory stall split from captured data. Backend stall percentage used as Tier 2/3 covariate in SE-7 tail attribution (Mann-Whitney U, Shapley decomposition). Missing: cache miss counters, branch mispredictions, L1D miss attribution. | Adapt |

#### Analysis

| Capability | Enables | Status | Rationale (see Appendix B for detail) | Strategy |
|-----------|---------|--------|---------------------------------------|----------|
| Latency distribution | P1; SE-1,6,7,11 | Yes | Per-window latency with full distribution stats (median, P95, P99, std) and CDF plots for visual analysis. | Implemented |
| Deadline analysis | P1; SE-1 | Partial | Per-window deadline tracking and miss rate computation via `cortex check-deadline` CLI; missing root-cause correlation with platform state. | Adapt |
| Analysis/reporting | P1; SE-11 | Yes | `cortex analyze` generates latency/CDF/throughput plots (PNG/PDF/SVG), SUMMARY.md, and full statistics with execution environment. | Implemented |
| Comparative analysis | P3; SE-2, HE-1 | Yes | `cortex compare` with Welch's t-test, Cohen's d (with effect size labels), percentile comparison (P50/P95/P99), and practical significance verdict (IMPROVED/REGRESSED/NEGLIGIBLE/NOISE/LOW_N) with sample-size guard. CDF overlay and bar chart visualizations. | Implemented |
| Latency decomposition | P4,P5; SE-5,7 | Yes | 3-step latency analysis: `cortex predict` (pre-benchmark Roofline regime classification—compute vs memory bound), `cortex run` (benchmark with PMU telemetry), `cortex decompose` (post-hoc characterization fitting measured PMU data to distribution landmarks, IPC/frequency/stall decomposition, and provenance tracking). SE-7 tail-latency attribution integrated: `attribute_tail()` provides three-tier analysis—Tier 1 (P99/P50 diagnostic ratios, noop-normalized ratio, platform/algorithmic verdict), Tier 2 (Mann-Whitney U cohort comparison across cpu_freq_mhz, osnoise_total_ns, backend_stall_pct covariates; KS frequency stratification), Tier 3 (Shapley R² variance decomposition attributing latency variance to platform covariates vs algorithmic residual). | Innovate |
| Diagnostic framework | P4,P5; SE-5,7 | Partial | `cortex decompose` compares Roofline floor against measured distribution to reveal overhead. PMU backend stall decomposition splits kernel time into compute vs memory stall. SE-7 tail attribution adds statistical diagnostics: Mann-Whitney U tests identify covariates elevated in tail windows (p < 0.05), KS test detects frequency-stratified P99 differences, Shapley R² decomposes latency variance into platform covariates vs algorithmic residual. Missing: automated predicted-vs-actual residual categorization, actionable optimization recommendations. | Adapt |
| Mandatory reporting | P3,P4; SE-7, HE-1 | No | Capture compiler version/flags, CPU governor, and frequency in telemetry records for reproducible cross-platform comparison. | Adapt |
